# Trello-Kanban board db design

requirements >>

1. users can sign up 
2. users can create one or more board. either public or private. we also want to know when the board was created
3. a board can have multiple column, and they can be moved, renamed, deleted
4. a user can  add card to a list on a board. cards can be moved to a different list. it can have a name and desc.
5. comments can be added to card by users who can access the board
6. ability to add checklist item to  a card
7. users can be added as members to a card to see updates
8. any activity on a card should be tracked 
9. a card can be archived. 
10. a user can add label to the card, edit label names and colours and create labels
11. A card can have a due date and a reminder date and time
12. a user can attach files and pictures to the card
